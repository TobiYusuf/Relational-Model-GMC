Relational Model for Hotel

Description of Relationships

Hotel: Hotel_Id as the primary key and Hotel_Name as an attribute.

Type Table: Contains Type_Id as the primary key and Type_Name as an attribute.

Category Table: Contains Category_Id as the primary key, Category_Name, Price, and Beds_numbers as attributes.

Employee Table: Contains Employee_Id as the primary key, Employee_Name, Employee_Speciality as attributes, and Hotel_Id as a foreign key referencing the Hotel table.

Leads Table: Contains Employee_Id and Room_Id as primary keys and foreign keys, referencing the Employee and Room tables, respectively.
